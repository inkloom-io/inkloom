# Getting Started

Follow these steps to create, write, and deploy your first documentation site.

<Steps>
<Step title="Install the CLI">

Install the InkLoom CLI globally using your preferred package manager.

<Tabs>
<Tab title="npm">

```bash
npm install -g @inkloom/cli
```

</Tab>

<Tab title="pnpm">

```bash
pnpm add -g @inkloom/cli
```

</Tab>

<Tab title="yarn">

```bash
yarn global add @inkloom/cli
```

</Tab>
</Tabs>

Verify the installation by running `inkloom --version`.

</Step>

<Step title="Authenticate">

Log in with your API key from the InkLoom dashboard (**Settings → API Keys**).

```bash
inkloom auth login
```

</Step>

<Step title="Create a project">

Create a new documentation project and note the returned project ID.

```bash
inkloom projects create --name "My Docs"
```

</Step>

<Step title="Push and deploy">

Push your local MDX files and deploy to production.

<CodeGroup>
```bash title="Push pages"
inkloom pages push <project-id> --dir ./docs --publish
```

```bash title="Deploy"
inkloom deploy <project-id> --production --wait
```
</CodeGroup>

Your documentation site is now live.

</Step>
</Steps>

<Callout type="tip" title="Use dry-run first">

Run `inkloom pages push <project-id> --dir ./docs --dry-run` to preview changes before pushing for real.

</Callout>
